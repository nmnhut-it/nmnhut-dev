How Computers See

A machine perceives the world as data — differently from you. Before any code, let's meet that gap. It's the hook for the whole course.

This is the on-ramp. No Python yet. Goal: make students feel that the machine gets handed data, not meaning. Everything later is turning a problem into data, then writing exact rules over it.

You Just Knew

A picture ? ? ? "triangle" or "rectangle"

You knew the answer instantly — without doing any "work." But what does a machine actually get handed?

Show the shape; everyone reads "triangle" in a fraction of a second. Ask how they knew. They can't really explain it — they just saw it. That effortless seeing is exactly what the machine cannot do. The empty middle box is what this whole unit is about.

The Machine Gets No "Triangle"

A machine has no eyes and no idea of "shape." A camera hands it something much plainer: a grid of tiny light readings.

Light bounces off the shape The camera splits it into tiny squares Each square reports 1 (shape) or 0 (background)

"See" is the wrong word. The machine measures.

Draw the pipeline: light to camera to a grid of numbers. The key reframe — the machine never receives the word "triangle," it receives measurements. One number per tiny square. That grid is the machine's entire world.

A Picture Is a Grid of 0s and 1s

10000 11000 11100 11110 11111

Read it row by row. Where is the "triangle"? It isn't labelled anywhere — it's just 1 = shape, 0 = background.

Same triangle, now as the machine holds it: a matrix of bits. Have them trace the filled cells. Point out the slanted edge is already turning into a staircase of 1s — plant that seed for later. This grid is the real input for the rest of the unit.

Our Job: Turn Seeing Into a Rule

Grid of 0s and 1s An exact rule "triangle" or "rectangle"

Only two answers are allowed, so we lean on the one thing that tells the shapes apart:

Triangle

3 corners

Rectangle

4 corners

Plan

find every corner, then count

Name the human strategy before encoding it: triangles have 3 corners, rectangles 4. If we can count corners, we can decide. But "count the corners" is still a human phrase — next we have to make "corner" checkable on a grid of numbers.

What Is a Corner, to the Machine?

A person just points. The machine can only look at one cell and its neighbours. So we spell it out:

A corner is a filled cell (1) with empty space (0) on two sides that meet at a right angle — an L-shape of emptiness.
000 011 011

Above is empty and left is empty — two perpendicular sides. That makes the red cell a corner.

Turn the fuzzy word "corner" into a local, checkable rule: filled cell with empty on two perpendicular sides. Have them point at the two empty directions. This is the first real taste of "turn a human judgment into an exact rule over data."

Run the Rule: Rectangle

11111 11111 11111 11111 11111
4 corners → rectangle

A cell in the middle of an edge is empty on only one side, so it fails the rule. Only the 4 true corners pass. It works!

A real win — the encoded rule succeeds. Ask why an edge-middle cell doesn't count (empty on one side, not two perpendicular). Exactly 4 corners means rectangle. Let them enjoy it, because the next shape breaks it.

Red = the 3 real cornersAmber = stair-step corners the rule also found

The exact same, honest rule now counts 6. The slanted edge is a staircase of 1s, and every step really does have empty space on two perpendicular sides — so each step passes the corner test. The rule isn't buggy; the machine simply doesn't see a smooth line.

Quick Check

Why did the machine count 6 corners on the triangle?

Answer: the third. The rule ran honestly. To the machine a slope is a staircase of bits, and each stair-step is a filled cell with empty space on two perpendicular sides — a corner by our own definition. The model was too simple, not the machine.

That Gap Is the Whole Point

You see a smooth slanted line. The machine sees stairs — and every stair-step looks like a little corner. Same picture, different perception.
Human

"obviously 3 corners"

Machine

a grid where a slope is a staircase

Our job

write a rule that survives the machine's view

The machine doesn't see what you see. Closing that gap — turning what you notice into rules a machine can run — is what this whole course is about.

Land the thesis. The wrong count is nobody's fault: the rule was honest, our model was too simple, and the machine perceives differently. The fix is a smarter model, not a smarter machine. Real computer vision spends its life closing exactly this gap.

The Modeling Loop

Represent

hold the input as data (a grid of 0/1)

Rule

exact steps from data to answer (count corners)

Break & refine

find the edge case (the staircase), fix the model

We just did all three. You'll run this same loop on numbers, text, lists, and maps — all year.

Name the moves once, don't chant them: represent the input as data, write an exact rule, then break it on an edge case and refine. This unit ran the loop with an image; every future unit reruns it with a new kind of data.

Recap & Next

  • A machine is handed data (a grid of numbers), never meaning.
  • To make it decide, you turn what you notice into an exact rule.
  • Rules meet edge cases — you break them and refine.
