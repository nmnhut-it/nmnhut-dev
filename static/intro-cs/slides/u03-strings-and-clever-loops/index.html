Strings & Clever Loops

Text is a row of symbols the machine can hold and check — one character at a time. Then a smarter way to search: halve the range. Based on MIT 6.0001, Lecture 3.

Unit 2 gave us choices and loops. Unit 3 turns loops loose on text, then on searching. Keep the thread visible: to a computer, text is just symbols in order — it checks symbols, never meaning.

Text Is a Sequence of Symbols

You see
the word "cat" The machine sees
'c', 'a', 't' — in that order

The computer checks symbols, not meaning. "cat" is just three characters in a row — it never knows it's an animal.

Representation thread for the whole unit. A string is symbols in order: letters, digits, spaces, punctuation. The machine compares characters; it has no idea what the word means. That's why we can loop over it like a list of boxes.

How Long Is It? len()

word = "cat"
len(word)     # 3   <- counts the characters
len("hi you") # 6   <- the space counts too!
len("")       # 0   <- empty string, nothing inside

A string lives inside quotation marks. len is a ready-made function that counts every character — spaces and punctuation included.

len is our first tool on strings. Stress: spaces are characters too, so "hi you" is 6. The empty string "" has length 0. len(word) is an expression, so it has a value you can print or store.

Reach One Character — Indexing

s = "PYTHON"
s[0]   # 'P'   <- counting starts at ZERO, not one
s[1]   # 'Y'
s[-1]  # 'N'   <- negative counts from the end
s[-2]  # 'O'
s[99]  # IndexError: string index out of range

Predict s[2] before you run it. Remember: the first character is position 0, and -1 is the last one.

Zero-based indexing trips everyone at first. s[0] is 'P', so s[2] is 'T'. Negative indices count back from the end; s[-1] is the last character. Going past the end doesn't crash the world — Python just raises an IndexError and names the line.

Grab a Piece — Slicing s[a:b]

s = "abcdefgh"   # index:  0 1 2 3 4 5 6 7
s[2:5]   # 'cde'   <- from 2 up to (not including) 5
s[:3]    # 'abc'   <- leave start blank = from 0
s[5:]    # 'fgh'   <- leave stop blank = to the end
s[:]     # 'abcdefgh'  <- the whole string

s[a:b] gives you the characters from a up to but not including b. Leave a number out and Python fills in a sensible default.

The classic gotcha: the stop is exclusive. s[2:5] is c,d,e — not f. Blank start means 0; blank stop means the end. When unsure, type it in the interpreter and look. This is the same start/stop idea as range from Unit 2.

Glue Strings Together — +

first = "Ada"
last  = "Lovelace"
name = first + " " + last   # 'Ada Lovelace'
print(name)

print("ha" * 3)             # 'hahaha'  <- repeat with *

The + sign joins two strings into a new one. (Don't forget the space in the middle!) And * repeats a string.

Concatenation with +. Common beginner miss: forgetting the " " so you get 'AdaLovelace'. * repeats. Note + on strings joins, while + on numbers adds — same symbol, the type decides what happens.

Walk Through Each Character

s = "banana"
count = 0
for char in s:          # char becomes 'b', then 'a', then 'n', ...
    if char == "a":
        count = count + 1
print(count)            # 3   <- three a's

Trace it: what is char on each pass, and what is count at the end?

A for loop walks over any sequence, including a string. char is bound to each character in turn — no s[index] bookkeeping needed. This "for char in s" reads like English. Trace it out loud: char = b,a,n,a,n,a; count ticks up on each a to 3.

Strings Can't Be Edited — Immutable

s = "hello"
s[0] = "y"          # ERROR -- can't change a character in place
s = "y" + s[1:]     # OK -- builds a NEW string "yello"
print(s)            # yello

You can never edit a string in place. To "change" it, you build a brand-new string and give it the same name. The old "hello" is simply left behind.

Immutability, gently. The object is frozen; s[0] = "y" fails. The fix rebuilds: "y" + s[1:] makes a fresh "yello" and rebinds s. Nothing edited the original. This matters again for lists in Unit 5, which CAN be edited.

A Slow Way to Search: Guess & Check

# Is 8 a perfect cube? Try 0, 1, 2, ... in order.
cube = 8
guess = 0
while guess ** 3 < cube:
    guess = guess + 1          # crawl upward one at a time
print(guess)                   # 2   <- because 2*2*2 = 8

Try every candidate in order until one fits. Simple and correct — but if the answer is big, you crawl through a lot of numbers.

Guess-and-check = try every value in order. Fine for small numbers; for a huge target it's slow because it steps one at a time. That slowness sets up why we want something cleverer: bisection.

A Clever Way: Halve the Range

I'm thinking of a number from 1 to 100. Guess, and I'll say "higher" or "lower".
50? higher → 75? lower → 62? higher → 68? ... found in about 7 guesses, not 100.

Guess the middle. Each answer throws away half the numbers still left. That's bisection — cutting the range in two every time.

The guessing game everyone knows. Always guess the middle; each "higher/lower" halves what's left. 100 numbers falls to 1 in about 7 halvings. Contrast with guess-and-check crawling 1,2,3,... The bigger the range, the bigger the win.

Bisection in Code

target = 68
low = 1
high = 100
guess = (low + high) // 2
while guess != target:
    if guess < target:
        low = guess + 1        # too low: raise the floor
    else:
        high = guess - 1       # too high: lower the ceiling
    guess = (low + high) // 2  # new middle
print("found", guess)

Keep a low/high bracket, test the middle, drop the half that can't hold the answer, repeat.

Same idea as the game, in code. low and high bracket the answer; the midpoint is tested; we move whichever end can't contain the target. Don't sweat the math — the point is: halving beats crawling. This mirrors the L3 bisection square-root exercise.

Quick Check

With s = "abcdefgh", what is s[3:6]?

The trap is the exclusive stop. s[3:6] takes indices 3, 4, 5 — d, e, f — and stops before 6, so no g. From the L3 quiz.

Recap & Next

  • A string is a sequence of symbols: len(s), index s[i] (from 0), slice s[a:b], join with +, loop with for char in s.
  • Strings are immutable — you build a new one, you never edit in place.
  • Guess-and-check crawls; bisection halves the range and finds the answer far faster.
