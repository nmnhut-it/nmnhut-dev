Choices & Repetition

Teach a program to decide and to repeat. Two small ideas that turn straight-line code into anything. Based on MIT 6.0001, Lecture 2.

Booleans: Yes/No Data

age = 15
age >= 13     # True
age == 18     # False   <- two equals means "is it equal?"
age != 18     # True    <- not equal

A comparison is a question. Its answer is a bool: just True or False — the simplest data of all.

Represent thread: a bool is a yes/no fact stored as data. Stress == compares, = stores. Predict each line, then run.

Combining Tests: and / or / not

sunny = True
warm  = False

sunny and warm    # False  <- needs BOTH true
sunny or warm     # True   <- needs EITHER true
not sunny         # False  <- flips it

and is picky (both), or is easy (either), not flips. Each still gives back a bool.

Read them as English. Predict the three results before revealing. These build the compound conditions branches will use.

if / else: One Decision

temp = 30
if temp > 25:
    print("wear shorts")     # runs only if True
else:
    print("bring a jacket")  # runs only if False
print("let's go")           # runs either way

The indentation (4 spaces) is how Python knows which lines belong to the choice.

Predict output for temp = 30, then change to 10 and re-predict. Exactly one branch runs; the un-indented line always runs. Indentation is not decoration — it is the grammar.

if / elif / else: First True Wins

score = 82
if score >= 90:
    print("A")
elif score >= 80:
    print("B")     # <- first True, so this runs
elif score >= 70:
    print("C")
else:
    print("try again")

Python checks top to bottom and takes the first true branch — then skips the rest, even if a later one is also true.

elif = "else if". Trace score = 82: 90 fails, 80 passes, prints B, stops. Order matters. Ask what prints for score = 95.

The while Loop (and the one that never stops)

n = 1               # 1. START value
while n <= 3:       # 2. CONDITION (a bool)
    print(n)        #    prints 1, then 2, then 3
    n = n + 1       # 3. UPDATE  <- forget this = forever!

A counting while needs a start and an update. Drop the update and the condition stays true — an infinite loop (press Ctrl-C to stop).

The loop keeps state: n changes each pass, and that changing value ends the loop. Delete n = n + 1 live to show the runaway, then Ctrl-C.

The for Loop + range()

for n in range(3):        # 0, 1, 2   (stops BEFORE 3)
    print(n)

range(2, 6)               # 2, 3, 4, 5
range(0, 10, 2)           # 0, 2, 4, 6, 8   (step by 2)

A for loop counts for you — no start or update to forget. range(start, stop, step) stops just before stop.

range(3) is 0,1,2 — the off-by-one to internalise. Predict range(2,6) and range(0,10,2) before running. for = known count; while = unknown count.

Early Exit: break & continue

for n in range(10):
    if n == 5:
        break        # leave the loop right now -> stops at 4
    print(n)         # 0, 1, 2, 3, 4

for n in range(5):
    if n % 2 == 0:
        continue     # skip THIS pass, keep going
    print(n)         # 1, 3

break quits the loop; continue skips to the next round.

Trace both. break = done with the loop; continue = done with this pass only. n % 2 == 0 tests even. Predict the printed numbers first.

Worked Example: Add 1 to n

n = 5
total = 0                    # running sum starts empty
for k in range(1, n + 1):    # k = 1, 2, 3, 4, 5
    total = total + k        # keep adding into total
print(total)                 # 15

Trace it: after each pass, what is k, and what is total? The loop keeps state that changes.

Fill a table: pass, k, total (1/1, 2/3, 3/6, 4/10, 5/15). range(1, n+1) reaches n because stop is excluded. This is the sum_1_to_n exercise.

Quick Check

Which expression is True?

C is True or False, which is True. A is not True = False; B is True and False = False; D is False. Read each aloud in English.

Recap & Next

  • A bool is yes/no data; comparisons and and/or/not produce it.
  • Choose with if / elif / else — the first true branch wins, marked by indentation.
  • Repeat with while (needs start + update) and for (uses range); break/continue steer the loop.
