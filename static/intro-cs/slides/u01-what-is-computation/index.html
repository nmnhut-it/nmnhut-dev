Unit 1 · High school CS in Python

What is Computation?

No slides-first lecture. You'll write and run real Python on these slides in the next two minutes — right here in the browser.

Hook-first unit. Learners run code before we define anything. Optional 3-min opener: "be the computer" — a student gives you literal instructions to draw a shape or make a sandwich; follow them maliciously literally until it fails. Land: "computers do EXACTLY what you say." Then jump to slide 2.

Predict, then Run

Guess each line's output before you press Run. Then run it.


          

          
First run loads Python in the browser — a few seconds, once.

          
The surprise IS the lesson. "7" + "7" is 77 (gluing text); 7 + 7 is 14 (adding numbers). "ha" * 20 spams hahaha. They just met strings vs numbers — the machine sees "7" and 7 as different DATA. Have them predict out loud first.

        


        

          
Why Two Answers?

          

            
"7"
text (a string) — the machine sees a symbol

            
7
a number (an int) — the machine can do math

          

          
Same key on your keyboard, two different kinds of data. Choosing how to store something as data is the whole game.

          
This is the representation thread landing on the very first surprise. Quotes = text; no quotes = number. "+" glues text but adds numbers. That's Unit 0's idea in code.

        


        

          
Make It Talk to You

          
Run it, type your name when it asks. Then change the message and run again.

          

            
            
            

          

          
A box pops up for your name. That's input().

          
Instant personalization = ownership. name is a VARIABLE holding text you typed. Dare them: make it print the name 100 times, or change the greeting. First variable, first input, first "it talked to ME," under 5 minutes.

        


        

          
The One Big Rule

          
A computer does exactly what you tell it — no more, no less. Fast, not smart.

          
It can't guess what you meant. If the output is wrong, the instructions were wrong. That's not failure — that's the job.

          
Now that they've run code, name the big idea (CS50's PB&J point). Tie to the "be the computer" opener if you did it. This reframes bugs as normal, not personal.

        


        

          
Draw With a Loop

          
Run it. Then change "*" to "🔥", and change the 6 to 11. Run again.

          

            
            
            

          

          
A for loop repeats a line. You'll learn loops properly in Unit 2 — for now, watch the shape grow.

          
Loops sell themselves when you SEE the triangle grow. Don't teach range() formally yet — let them tinker. Changing one number changes the picture = the type-in-and-modify tradition (Ahl/Sweigart).

        


        

          
Numbers Are Data

          
Predict each result, then run. Try changing the numbers.

          

            
            
            

          

          
A single / always gives a float (3.5). // is floor division, % is remainder (huge later: even/odd, digits), ** is power. A whole number is an int, a decimal is a float.

        


        

          
Names for Values

          
Change the price or quantity and re-run.

          

            
            
            

          

          
The = sign stores a value under a name so you can reuse it.

          
A variable is a labeled box in memory. = means "store this," not math equality. Descriptive names make code readable.

        


        

          
It Only Does What You Say

          

            
            
            

          

          
Changing radius does not update area. The computer never re-figures things for you.

          
The lecture's punchline, now runnable. area was computed once and stays put. This kills a huge class of beginner confusion — run it and let the "still 12.56636" surprise land.

        


        

          
Types, and Switching Between Them

          

            
            
            

          

          
Every value has a type that decides what you can do with it.

          
int() truncates toward zero: 3.9 -> 3, not 4. str() turns a number into text so you can glue it. bool (True/False) shows up in Unit 2.
